CHH blog shell (typical /blog/*.html page): component-init.js (before bundles) → main.js (defer: nav, footer, optional CSS/Bootstrap) → product-components-bundle.js → optional Converge pixel + inline cvg bootstrap. Body uses article-body and a main column <section class="w-60 mx-auto my-4">. Article chrome is mostly static HTML (breadcrumb, category badge, title, author, share row); no fold.js (HPI fold). Related cards use the same script filename as HPI: article-registry.js then related-articles.js, which exports createAdditionalReading into #additional-reading-section. Injected pieces: createProductShowcase inside .product-showcase-wrapper (often with addShopButton), then that block. Use initComponentsWhenReady(..., { waitForProduct: true, waitForArticle: false, waitForAds: false }) so the callback does not wait for HPI-only article globals.
